## Core Idea

**Summary:**

Robert C. Martin argues that code should be written for humans to read. Readable, well-structured code reduces bugs, improves maintenance, and speeds future work.

The book presents practical principles and examples for naming, functions, classes, and test-driven design.

---

## Clean Code Rules

- Use meaningful names.
- Keep functions small and focused.
- Avoid duplication.
- Make code behavior explicit.
- Prefer readability over clever shortcuts.

---

## Practical Checklist

- [ ] Choose clear names for variables and functions.
- [ ] Keep each function short and single-purpose.
- [ ] Remove duplicate logic.
- [ ] Write tests to document expected behavior.
- [ ] Refactor continuously to keep the codebase clean.
