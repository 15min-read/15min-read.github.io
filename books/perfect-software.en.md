## Core Thesis

**Summary:**

Roger Pressman argues that perfect software is not about zero defects, but about creating a disciplined process that maximizes reliability, usability, and maintainability.

Quality emerges from design, development practices, testing, and process maturity.

---

## Quality Practices

- Define requirements clearly.
- Follow structured design discipline.
- Use continuous testing and review.
- Treat quality as a team priority.

---

## Practical Checklist

- [ ] Clarify what the software must do.
- [ ] Apply consistent design principles.
- [ ] Review code and tests often.
- [ ] Build quality checks into each phase.

## Notes

The key value is the idea that software quality is systemic. Defects are symptoms of weak processes, not just careless coding.
